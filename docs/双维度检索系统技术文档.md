# Hyper-RAG 双维度检索系统技术文档

## 1. 系统概述

Hyper-RAG 双维度检索系统是一种基于异质单纯复形（Heterogeneous Simplicial Complex, HSC）的知识图谱检索引擎，同时融合**拓扑结构维度**和**语义内容维度**进行综合检索，旨在解决传统 RAG 系统中检索结果缺乏结构连贯性和语义相关性的问题。

### 核心设计理念

| 维度 | 检索依据 | 提供能力 | 发现方式 |
|------|----------|----------|----------|
| **拓扑结构维度** | 图谱结构关系、连接方式、层级组织 | 结构连贯性 | 与查询实体在图结构上相连的复形 |
| **语义内容维度** | 文本语义相似度、上下文理解、概念关联 | 内容相关性 | 内容上与查询语义相关的复形，即使拓扑上不直接相连 |

两个维度互补：拓扑维度确保检索结果在知识图谱中结构可达，语义维度确保即使拓扑不直接相连但语义相关的内容也能被检索到。

## 3. 核心数据结构

### 3.1 异质单纯复形（HSC）

**文件**：`_simplicial_complex.py`

HSC 是整个检索系统的基础数据结构，将知识图谱中的实体和关系组织为不同维度的单纯形：

| 维度 | 对应 | 示例 |
|------|------|------|
| 0-单纯形 | 实体（节点） | `"CONTRACT_A"` |
| 1-单纯形 | 关系（边） | `("CONTRACT_A", "PARTY_B")` |
| 2-单纯形 | 三元组（面） | `("CONTRACT_A", "PARTY_B", "TERMS")` |
| k-单纯形 | 高阶结构 | k+1个实体的组合 |

**核心属性**：

```python
class HeterogeneousSimplicialComplex:
    nodes: dict           # {node_id: {type, vector}}
    simplices: dict       # {simplex_id: simplex_data}
    B_matrices: dict      # {dim: 关联矩阵Bk}
    L_matrices: dict      # {dim: Hodge Laplacian矩阵Lk}
    _data_version: int    # 数据版本号（缓存一致性）
    _laplacian_version: int  # Laplacian版本号
```

**版本管理机制**：

- `invalidate_data()`：数据变更后递增版本号，使 Laplacian 缓存失效
- `needs_laplacian_rebuild()`：检查缓存一致性
- `mark_laplacian_current()`：Laplacian 计算完成后标记版本一致

### 3.2 单纯形数据格式

```python
simplex_data = {
    'id': str,              # 单纯形ID
    'dimension': int,       # 维度（0/1/2/...）
    'entities': list,       # 实体列表
    'nodes': list,          # 节点列表（同entities）
    'type': str,            # 类型
    'entity_type': str,     # 实体类型
    'description': str,     # 描述
    'boundary': list,       # 边界（低维子单纯形ID列表）
    'coboundary': list,     # 上边界（包含此单纯形的高维单纯形ID列表）
    'importance': float,    # 重要性（0~1）
    'frequency': int,       # 出现频率
    'source': str,          # 来源文本块ID（<SEP>分隔）
    'level_hg': float,      # 显著性级别
}
```

## 4. 检索算法详解

### 4.1 阶段1：查询预处理

#### 4.1.1 实体/关系提取

**函数**：`_extract_query_entities(query, global_config)`

**流程**：
1. 使用 LLM 从查询文本中提取实体和关系
2. 解析 LLM 返回的 JSON（支持 ```json 包裹）
3. 标准化实体名称（`normalize_entity_name`：大写 + 去空格）
4. 关系数量上限5条，优先保留高维关系

**输出**：
```python
extracted_entities = [{'type': 'Entity', 'name': 'CONTRACT_A', 'description': ''}]
extracted_relations = [{'entities': ['A', 'B'], 'description': '...', 'dimension': 1}]
```

#### 4.1.2 HSC 构建

**函数**：`_build_hsc_from_storage(simplex_storage)`

**流程**：
1. 优先从缓存加载 Laplacian 矩阵和复形数据
2. 缓存失败则从 `simplex_storage` 全量重建
3. 重建时自动补全 `coboundary` 和 `boundary` 关系
4. 构建 Hodge Laplacian 矩阵

#### 4.1.3 实体匹配

**函数**：`_match_entities_to_hsc(extracted_entities, hsc, simplex_storage, entities_vdb)`

**四级匹配策略**：

| 优先级 | 策略 | 阈值 | 说明 |
|--------|------|------|------|
| 1 | 精确匹配 | 1.0 | 标准化后名称完全一致 |
| 2 | 子串匹配 | 0.6~1.0 | 包含关系 + 排除反义词对 |
| 3 | VDB语义匹配 | 0.3~0.5 | 向量数据库余弦相似度 |
| 4 | 虚拟节点 | 0.1 | 创建临时节点，查询后自动清理 |

**反义词排除对**：`("SECURED", "UNSECURED")`, `("BINDING", "NON-BINDING")`, `("EXEMPT", "NON-EXEMPT")`

### 4.2 阶段2：拓扑结构维度检索

#### 4.2.1 分发（Parallel Ignition）

**维度A - 实体点火**：
- 将匹配到的查询实体作为种子节点（0-单纯形）
- 通过 `get_upper_adjacent` 查找每个实体的上边界复形

**维度B - 关系模式匹配**：
- 函数：`_match_relations_to_simplices(enhanced_relations)`
- 通过倒排索引 `_entity_inverted_index` 快速查找包含关系实体的复形
- 使用自适应匹配阈值 `_calculate_match_score`
- 部分匹配关系降权：`adjusted_score = score * match_ratio`

#### 4.2.2 扩散（Topological Diffusion）

**函数**：`_execute_diffusion(seed_nodes, seed_edge_ids, seed_high_dim_simplices, relation_to_simplices)`

**核心算法**：基于 Hodge Laplacian 的拓扑能量扩散

```
Lk = Bk^T × Bk + Bk+1 × Bk+1^T    (Hodge Laplacian)
x(t+1) = x(t) - α × Lk × x(t)      (热扩散方程)
```

**安全机制**：
- 基于 Gershgorin 圆定理估计 Laplacian 谱半径上界
- 确保 `α < 1.5 / λ_max`，防止扩散发散
- 数值膨胀早期干预：`max_val > 1e3` 时归一化
- NaN/Inf 检测：出现数值不稳定立即停止扩散

**按维度扩散**：
- dim=0：节点扩散，发现与种子节点拓扑邻近的实体
- dim=1：边扩散，发现与种子边拓扑邻近的关系
- dim≥2：高维扩散，发现与种子复形拓扑邻近的高阶结构

#### 4.2.3 对撞（Coboundary Contraction）

**函数**：`_compute_coboundary_contraction(vertex_ids, type_filter)`

**核心逻辑**：维度A通过 B^T 矩阵向上辐射，寻找包含查询实体的面

1. 计算每个查询顶点的上边界（coboundary）
2. 求严格交集（所有查询顶点共同的上边界复形）
3. 对非交集候选计算语义相似度（使用嵌入缓存）
4. 自适应覆盖阈值过滤

**候选过滤**（`_filter_candidates`）：
- 严格交集复形直接保留
- 单顶点查询：按维度 + 语义相似度排序
- 多顶点查询：覆盖度 ≥ 1 或覆盖率 ≥ 自适应阈值 + 语义相似度 ≥ 自适应阈值

#### 4.2.4 拓扑内部融合

**函数**：`_fusion_dual_dimensions(candidate_simplices, common_coboundary, pattern_simplices, ...)`

**融合策略**：
1. 维度A得分：`calculate_simplex_score(data, vertex_ids, None, score_type='A')`
2. 维度B得分：`calculate_simplex_score(data, None, relation_to_simplices, score_type='B')`
3. 动态权重：`compute_dynamic_fusion_weights(vertex_quality, relation_quality)`
4. 加权融合：`weighted_score = weight_a × a_score + weight_b × b_score`
5. 自适应阈值：30分位数
6. 高维扩散结果融合：`combined_score = diffusion_score × coverage_count² × importance`

**双维度均失效回退**：当 `vertex_quality=0` 且 `relation_quality=0` 时，使用等权融合 + 扩散得分排序

### 4.3 阶段3：语义内容维度检索

#### 4.3.1 嵌入向量语义检索

**函数**：`_semantic_retrieve(query_text, top_k)`

**流程**：
1. 构建复形文本缓存（dim≥1 的复形：实体名 + 描述）
2. 获取查询嵌入（带缓存，LRU淘汰）
3. 批量获取复形嵌入（已缓存直接返回，未缓存批量计算）
4. 本地余弦相似度计算（无需API调用）
5. 最低相似度过滤（`SEMANTIC_MIN_SIMILARITY=0.3`）
6. Top-K 截断（`SEMANTIC_RETRIEVE_TOP_K=50`）

**嵌入缓存机制**：

| 缓存类型 | 作用 | 淘汰策略 |
|----------|------|----------|
| `_simplex_embedding_cache` | 复形文本嵌入向量 | 数据更新时全量清除 |
| `_simplex_text_cache` | 复形文本映射 | 数据更新时全量清除 |
| `_query_embedding_cache` | 查询文本嵌入向量 | LRU：超过100条清除最早50条 |

#### 4.3.2 向量数据库语义检索

**函数**：`_semantic_vector_retrieve(query, entities_vdb, relationships_vdb, hsc, top_k)`

**流程**：
1. **实体VDB搜索**：查询 → `entities_vdb.query` → 命中实体
2. **Coboundary扩展**：命中实体 → `coboundary` 列表 → 高维复形（衰减系数0.8）
3. **关系VDB搜索**：查询 → `relationships_vdb.query` → 命中关系
4. **关系实体扩展**：关系实体 → `coboundary` 列表 → 高维复形（衰减系数0.7）

**最低相似度过滤**：所有结果（含扩展结果）必须满足 `score ≥ SEMANTIC_MIN_SIMILARITY`

### 4.4 阶段4：双维度融合

#### 4.4.1 拓扑-语义融合

**函数**：`_fusion_topology_semantic(topology_simplices, topology_weighted_scores, semantic_scores, vertex_ids, coboundary_threshold)`

**融合步骤**：

1. **质量评估**：
   - 拓扑质量 = 0.5 × 覆盖率 + 0.3 × 得分集中度 + 0.2 × 维度丰富度
   - 语义质量 = 0.6 × 高相似度占比 + 0.4 × 分布质量

2. **动态权重计算**：
   ```python
   weight_topo, weight_sem = compute_topology_semantic_weights(topology_quality, semantic_quality)
   # 权重约束在 [TOPO_SEM_WEIGHT_MIN=0.2, TOPO_SEM_WEIGHT_MAX=0.8]
   ```

3. **归一化**：
   - 分位数归一化（95分位数），支持负值得分（先平移到非负区间）
   - 所有得分相同时返回均匀分布

4. **加权融合**：
   ```python
   final_score = weight_topo × topo_normalized + weight_sem × sem_normalized
   ```

5. **自适应阈值**：
   ```python
   adaptive_threshold = min(coboundary_threshold, p30)
   adaptive_threshold = max(adaptive_threshold, p10)
   ```

#### 4.4.2 语义补充

将语义检索发现但拓扑维度未覆盖的高相似度复形加入结果：

```python
adaptive_threshold = mean(sem_only_scores) + 0.5 × std(sem_only_scores)
adaptive_threshold = max(0.4, min(0.8, adaptive_threshold))
```

#### 4.4.3 补全策略

按优先级依次尝试：

| 优先级 | 策略 | 函数 | 说明 |
|--------|------|------|------|
| 1 | 维度提升 | `_try_elevation` | 将低维复形拼接成高维复形 |
| 2 | 拓扑链补全 | `find_topological_chain` | 多源BFS搜索连接路径 |
| 3 | Common Coboundary | `_compute_common_coboundary` | 查询顶点的共同上边界 |

**多源BFS拓扑链**（优化版）：
- 从所有查询顶点同时出发进行BFS
- 使用 `source_reachability` 字典跟踪每个节点被哪些源顶点可达
- 当节点被 ≥2 个不同源顶点可达时，标记为拓扑链节点
- 时间复杂度：O(n×(V+E))，优于原 O(n²×(V+E))

### 4.5 阶段5：后处理与排序

#### 4.5.1 种子注入

**函数**：`_inject_seed_simplices(completion_results, seed_node_ids, seed_edge_ids, diffused_node_scores)`

- 种子节点（0维）：查询实体对应的0维复形
- 种子复形（1维）：查询关系直接匹配的边复形
- 标记为 `is_seed=True`，在子复形移除和截断时受保护

#### 4.5.2 子复形移除

**函数**：`_remove_sub_simplices(completion_results)`

**核心原则**：当子复形的实体集合是父复形的子集时，移除子复形，将其 `source_id` 合并到父复形。

**特殊处理**：
- 孤立种子实体（无父复形包含）保留
- 父复形获得 `parent_bonus` 加分

#### 4.5.3 维度配额截断

当结果数超过 `max_simplices` 时：
- 高维复形（dim≥2）分配 60% 配额
- 低维复形（dim<2）分配 40% 配额
- 剩余配额按综合得分补充

#### 4.5.4 文本块收集

**函数**：`_collect_text_chunks(ranked_simplices, text_chunks_db, ..., retriever)`

**两阶段策略**：

**阶段1 - 拓扑筛选**：
- 从复形的 `source` 字段收集候选文本块ID
- 计算拓扑权重：`weight = 1.0 + is_seed×5 + dimension×2 + diffusion×5 + coboundary×0.5 + semantic×5`
- 按拓扑权重初排

**阶段2 - 语义重排**：
- 优先使用 retriever 的嵌入缓存计算语义相似度
- 缓存不可用时回退到 `compute_semantic_similarity`
- 嵌入计算失败时回退到关键词重叠度（含法律领域停用词过滤）
- 最终得分：`0.6 × 语义得分 + 0.4 × 拓扑得分`

**多样性控制**：限制同一 `source_type` 的 chunk 数量

### 4.6 阶段6：结果返回

#### 4.6.1 提示指令生成

将检索结果注入 LLM 提示，包含三层信息：

1. **[Retrieved Entities]**：匹配到的实体（名称 + 类型 + 描述）
2. **[Retrieved Relations]**：识别到的关系（实体对 + 描述）
3. **[Topological Structures]**：发现的拓扑结构（维度 + 类型 + 实体 + 描述）

#### 4.6.2 虚拟节点清理

**函数**：`virtual_node_scope(hsc, query_vertices)`

使用异步上下文管理器确保虚拟节点在查询结束后自动清理：

```python
async with virtual_node_scope(hsc, query_vertices) as virtual_node_ids:
    # 整个检索流程
    ...
# 虚拟节点在 async with 块结束时自动清理
# 即使异常也会在 finally 块中清理
# 清理后自动失效 B_matrices 和 L_matrices 缓存
```

## 5. 配置参数系统

**文件**：`_config.py`

### 5.1 核心参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `TOPO_DIMENSION_WEIGHT` | 0.6 | 拓扑维度默认权重 |
| `SEMANTIC_DIMENSION_WEIGHT` | 0.4 | 语义维度默认权重 |
| `SEMANTIC_RETRIEVE_TOP_K` | 50 | 语义检索返回最大数 |
| `SEMANTIC_MIN_SIMILARITY` | 0.3 | 语义最低相似度阈值 |
| `TOPO_SEM_WEIGHT_MIN` | 0.2 | 拓扑-语义融合权重下限 |
| `TOPO_SEM_WEIGHT_MAX` | 0.8 | 拓扑-语义融合权重上限 |
| `SEMANTIC_RERANK_WEIGHT` | 0.6 | 文本块语义重排权重 |
| `TOPO_RERANK_WEIGHT` | 0.4 | 文本块拓扑重排权重 |

## 7. 关键算法原理

### 7.1 Hodge Laplacian

Hodge Laplacian 是代数拓扑中的核心算子，用于描述单纯复形的拓扑结构：

```
Lk = Bk^T × Bk + Bk+1 × Bk+1^T
```

其中 `Bk` 是 k-维关联矩阵，描述 (k-1)-单纯形与 k-单纯形之间的边界关系。

**物理意义**：
- `Bk^T × Bk`：度量 k-单纯形的"边界一致性"（下邻接）
- `Bk+1 × Bk+1^T`：度量 k-单纯形的"上边界一致性"（上邻接）
- `Lk` 的零空间对应 k-维同调群，反映拓扑"洞"的数量

### 7.2 拓扑能量扩散

基于热扩散方程的拓扑能量传播：

```
x(t+1) = x(t) - α × Lk × x(t)
```

- `x(0)`：种子节点初始能量（查询实体=1，其他=0）
- `α`：扩散率，受谱半径约束确保收敛
- 扩散后能量高的节点与查询实体拓扑邻近

### 7.3 分位数归一化

用于融合前的得分归一化，避免极端值压缩：

```python
# 1. 平移到非负区间
shifted = {k: v - min_val for k, v in scores.items()}

# 2. 使用95分位数作为参考值（而非最大值）
reference = percentile(shifted_values, 95)

# 3. 归一化到[0, 1]
normalized = {k: min(v / reference, 1.0) for k, v in shifted.items()}
```

## 8. 性能优化机制

### 8.1 嵌入缓存

| 优化点 | 实现 | 效果 |
|--------|------|------|
| 复形嵌入缓存 | `_simplex_embedding_cache` | 首次查询后，后续查询仅计算新增复形嵌入 |
| 查询嵌入缓存 | `_query_embedding_cache` + LRU淘汰 | 同一查询嵌入只计算1次 |
| 文本缓存 | `_simplex_text_cache` | 避免每次查询重复构建复形文本列表 |
| Retriever复用 | `hsc._retriever` 绑定 | 嵌入缓存在同一HSC的多次查询间共享 |

**预估效果**：首次查询后，后续查询嵌入API调用从3-4次降低到0-1次。

### 8.2 倒排索引

| 优化点 | 实现 | 效果 |
|--------|------|------|
| 实体倒排索引 | `_entity_inverted_index` | O(1)查找包含指定实体的复形 |
| VDB扩展优化 | 通过 `coboundary` 列表扩展 | O(M×K) 替代 O(N×M) 全量扫描 |

### 8.3 多源BFS

| 优化点 | 实现 | 效果 |
|--------|------|------|
| 拓扑链搜索 | 多源BFS + source_reachability | O(n×(V+E)) 替代 O(n²×(V+E)) |

### 8.4 扩散安全机制

| 优化点 | 实现 | 效果 |
|--------|------|------|
| 谱半径估计 | Gershgorin圆定理采样 | 确保 α < 1.5/λ_max，防止发散 |
| 早期干预 | max_val > 1e3 时归一化 | 避免数值精度损失 |
| NaN/Inf检测 | 每步检查 | 立即停止不稳定的扩散 |

## 10. 错误处理与回退策略

| 场景 | 回退策略 |
|------|----------|
| LLM实体提取失败 | 正则表达式提取大写开头的词 |
| HSC缓存加载失败 | 从 simplex_storage 全量重建 |
| 实体精确匹配失败 | 子串匹配 → VDB语义匹配 → 虚拟节点 |
| 嵌入API调用失败 | 回退到 `compute_semantic_similarity` 全量计算 |
| 嵌入计算全部失败 | 回退到关键词重叠度（含停用词过滤） |
| 拓扑融合无结果 | 维度提升 → 拓扑链补全 → Common Coboundary |
| 双维度均失效 | 等权融合 + 扩散得分排序 |
| 文本块收集为空 | VDB直接查询文本块 |
| 扩散数值不稳定 | 停止扩散，使用已有得分 |
| 虚拟节点未清理 | 上下文管理器 finally 块保证清理 |
